# Narrative Archetypes

Use this file to select a narrative structure for the talk. The
structure must serve the core argument, not the other way around.

## Selection Rule

1. Read the core argument and evidence matrix from the interview.
2. Identify the strongest evidence and the central claim it supports.
3. Choose the archetype that places this evidence at the narrative
   climax.
4. If no archetype fits, design a custom arc but justify it in
   `talk_plan.md` and identify the climax explicitly.

## Archetype 1: Problem-Driven

```text
unsolved problem -> why it matters -> our approach -> key evidence -> implication
```

- Use when: the talk introduces a new solution to a known problem.
- Climax: the key evidence slide that demonstrates the solution works.
- Common in: group meetings, conference talks, thesis chapters.
- Caution: do not spend too long on background. The audience already
  knows the problem exists.

## Archetype 2: Discovery-Driven

```text
unexpected observation -> investigation method -> evidence chain -> revised understanding -> next step
```

- Use when: the talk presents a finding that changes a prior assumption.
- Climax: the evidence that disproves the old understanding.
- Common in: group meetings, seminars, progress updates.
- Caution: the `unexpected` framing must be honest. Do not manufacture
  surprise if the result was expected.

## Archetype 3: Method-Driven

```text
method limitation -> new method design -> validation -> application -> comparison with prior methods
```

- Use when: the talk introduces a new computational or experimental
  method.
- Climax: the validation result that proves the method is reliable.
- Common in: methodology-focused talks, thesis defenses, conference
  methods sessions.
- Caution: validation must use a baseline or benchmark. A method without
  comparison is not a scientific contribution.

## Archetype 4: Comparison-Driven

```text
conflicting claims -> systematic comparison -> evidence table -> resolution -> remaining gap
```

- Use when: the talk reconciles contradictory results from multiple
  sources.
- Climax: the comparison table or figure that resolves the conflict.
- Common in: literature-review talks, group meeting reading reports,
  thesis introductions.
- Caution: do not default to paper-by-paper structure. Select claims
  and figures that serve the comparison, not papers that fill time.

## Archetype 5: Progress-Update

```text
project goal -> completed work -> current results -> challenges -> next plan
```

- Use when: the talk is a periodic progress report.
- Climax: the current results that advance the project goal.
- Common in: group meeting progress updates, milestone reviews.
- Caution: avoid turning every completed task into a separate section.
  Group results by scientific theme, not by task list.

## Custom Archetype

If none of the above fits:

1. Write the section sequence in `talk_plan.md`.
2. Identify the climax explicitly.
3. Justify why a custom structure serves the core argument better than
   any standard archetype.
4. Ensure the arc has a beginning (context), a middle (evidence), and
   an end (implication). A flat list of topics is not a narrative arc.

## Anti-Patterns

- `Background -> Methods -> Results -> Conclusion` without a narrative
  arc. This is a thesis outline, not a talk structure.
- Paper-by-paper summary for a group meeting, unless explicitly
  requested. Select claims and figures, not papers.
- `Introduction -> Everything I Did -> Summary`. This is a progress
  report with no scientific argument.
- No climax identified. Every talk needs a moment where the audience
  understands why the work matters.
